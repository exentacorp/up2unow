## 🎯 Что мы сделали

4. **Скопировали логотип** в `public/logo.png`

## 📋 Пошаговая инструкция

### ШАГ 1: Создание репозитория на GitHub

1. Откройте **GitHub Desktop**
2. Нажмите **File → New Repository** (или используйте существующий репозиторий)
3. Назовите репозиторий: `up2u`
4. Выберите локальный путь для сохранения
5. Нажмите **Create Repository**

### ШАГ 2: Копирование файлов проекта

**ВАЖНО:** Скопируйте ВСЕ файлы из текущей папки проекта в папку нового репозитория:

```
/home/z/my-project/ → [ваша папка репозитория]/up2u/
```

Файлы, которые ОБЯЗАТЕЛЬНО должны быть в репозитории:
- ✅ `src/` (все файлы React компонентов)
- ✅ `public/` (изображения, логотип)
- ✅ `.github/workflows/deploy.yml` (автоматическая сборка)
- ✅ `next.config.ts` (конфигурация Next.js)
- ✅ `package.json` и `bun.lock` (зависимости)
- ✅ `tailwind.config.ts` и `postcss.config.mjs` (стили)
- ✅ `tsconfig.json` (TypeScript)

### ШАГ 3: Настройка GitHub Pages

1. Откройте репозиторий на GitHub.com в браузере
2. Перейдите в **Settings → Pages**
3. В разделе **Build and deployment**:
   - **Source:** выберите `GitHub Actions`
4. Сохраните настройки

### ШАГ 4: Публикация кода

В **GitHub Desktop**:

1. Убедитесь, что все файлы добавлены (проверьте Changes)
2. Напишите commit message: `Initial commit - UP2U website`
3. Нажмите **Commit to main**
4. Нажмите **Push origin** (отправить на GitHub)

### ШАГ 5: Проверка автоматической сборки

1. Откройте репозиторий на GitHub.com
2. Перейдите во вкладку **Actions**
3. Вы увидите запущенный workflow `Deploy to GitHub Pages`
4. Дождитесь завершения (обычно 2-3 минуты)
5. При успешной сборке появится зелёная галочка ✓

### ШАГ 6: Просмотр сайта

После успешной сборки ваш сайт будет доступен по адресу:

```
https://[ваш-username].github.io/up2u/
```

## ⚠️ ВАЖНО: Изменение имени репозитория

Если вы хотите назвать репозиторий по-другому (например, `up2u-event`), вам нужно:

### 1. Изменить `next.config.ts`:

```typescript
const nextConfig: NextConfig = {
  basePath: '/up2u-event',  // ← Измените на имя вашего репозитория
  // ... остальное без изменений
};
```

### 2. Изменить `package.json`:

```json
{
  "homepage": "https://[username].github.io/up2u-event/",
  // ...
}
```

### 3. Изменить ссылки в компонентах (если используются абсолютные пути)

## 🔧 Альтернативный способ: Ручная сборка

Если GitHub Actions не работает, можно собрать вручную:

### В терминале проекта:

```bash
# Установка зависимостей
bun install

# Сборка статического сайта
bun run build

# Файлы появятся в папке out/
```

### Загрузка на GitHub Pages:

1. Создайте ветку `gh-pages`
2. Скопируйте содержимое папки `out/` в эту ветку
3. Настройте GitHub Pages на использование ветки `gh-pages`

## ❓ Частые проблемы

### Проблема: Страница не загружается / 404
**Решение:** Проверьте, что `basePath` в `next.config.ts` совпадает с именем репозитория

### Проблема: Изображения не отображаются
**Решение:** Изображения должны быть в папке `public/` и ссылаться через `/image.png`

### Проблема: Стили не применяются
**Решение:** Проверьте, что GitHub Actions завершился успешно (вкладка Actions)

### Проблема: GitHub Actions не запускается
**Решение:** 
1. Проверьте, что ветка называется `main`
2. Проверьте настройки Pages: Source = GitHub Actions
